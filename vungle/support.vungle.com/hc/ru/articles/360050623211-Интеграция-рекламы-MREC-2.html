Интеграция рекламы MREC

Vungle Agent
16/08/2021 17:39
Подписаться

Общие сведения

Начиная с Vungle Windows SDK 5.1.0+, Vungle поддерживает рекламные ролики MREC. MREC – это сокращение от «medium rectangle» (прямоугольник среднего размера). В отличие от рекламы внутри передачи, реклама MREC не требует полноэкранного воспроизведения. Как и баннерная реклама, рекламные ролики MREC являются прямоугольными окнами, которые могут быть расположены в любом месте приложения, обычно вверху или внизу экрана, позволяя пользователю продолжать взаимодействовать с приложением пока воспроизводится реклама. Размер контейнера для отображения рекламы MREC является принятым в отрасли стандартом: 300x250.

VungleAdControl обеспечивает отображение рекламы в форматах MREC, передавая контейнер в SDK Vungle посредством свойства AdConfig.AdContainer и управляя контентом Container.Content ведущего приложения.

Требования для рекламы MREC

  • Windows SDK 5.1.0+
  • UWP для Windows 10
  • Размещение рекламы MREC настраивается в панели управления Vungle
  • VungleAdControl шириной 300 и высотой 250

VungleAdControl

Инструменты MREC Vungle оптимизированы для отображения видео в контейнере размером 300 х 250. Убедитесь, что размер окна в VungleAdContrl не меньше 300 в ширину и 250 в высоту.

.xaml

<UI:VungleAdControl x:Name="embeddedControl" 
                    Width="300"
                    Height="250" />

Шаг 1. Выполните общую интеграцию

Для выполнения интеграции рекламы MREC в свое приложение для Windows начните с выполнения инструкций из статьи об общей интеграции . Настоящая статья содержит дополнительную информацию, и подразумевается, что общая интеграция уже выполнена.

Шаг 2. Загрузка, воспроизведение и закрытие рекламы MREC

Загрузка рекламы MREC

Загрузка рекламы MREC с помощью инструмента размещения MREC идентична загрузке полноэкранной рекламы.

.cs

sdkInstance.LoadAd("MREC_PLACEMENT_REFERENCE_ID")

Воспроизведение рекламы MREC

Для воспроизведения рекламы с помощью VungleAdControl требуется передача AppID, Placements и Placement. Вы также можете установить конфигурацию, при которой реклама MREC начинает воспроизводиться с отключенным звуком, задав параметр SoundEnabled как false.

Параметр Описание
AppID Идентификатор приложения должен быть таким же
Placement Контрольный идентификатор размещения для вашей рекламы MREC
Placements Укажите тот же контрольный идентификатор размещения, который вы указали для Placement (будет удален)
SoundEnabled Задайте как false, чтобы начинать воспроизведение с отключенным звуком (по желанию, по умолчанию установлено как true)

.cs

embeddedControl.AppID = "VUNGLE_APP_ID";
embeddedControl.Placements = "MREC_PLACEMENT_REFERENCE_ID";
embeddedControl.Placement = "MREC_PLACEMENT_REFERENCE_ID";

embeddedControl.SoundEnabled = false; // Optional (default = true)

await embeddedControl.PlayAdAsync();

Остановка отображения рекламы MREC

Для остановки отображения рекламы MREC вы можете использовать StopBannerAd в любое время.

.cs

embeddedAdControl.StopBannerAd();

Альтернативный способ: Простая интеграция рекламы MREC с помощью XAML

VungleAdControl предоставляет более простой способ интеграции рекламы MREC, если вам не требуются прослушиватели событий, или используется для контроля времени загрузки рекламы. После инициализации SDK он загрузит рекламу MREC автоматически, и вы можете выдать PlayAdAsync после завершения загрузки контента рекламы.

.xaml

<UI:VungleAdControl x:Name="embeddedControl" 
                    Width="300"
                    Height="250"
                    Placement="MREC-5209003"
                    Placements="MREC-5209003"
                    SoundEnabled="False" />

.cs

await embeddedControl.PlayAdAsync();
