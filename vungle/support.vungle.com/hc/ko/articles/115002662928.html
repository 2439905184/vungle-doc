View-Through Attribution 사용

TUNE

1. 클릭 URL 설정

  1. MAT 대시보드에 앱을 추가합니다.



  2. 최상의 결과를 얻으려면 TUNE에 Vungle을 처음 통합할 때 View Through Attribution을 설정하는 것이 좋습니다. 다시 사용하려면 어트리뷰션 설정 페이지에서 수행하십시오.

  3. Click을 선택하여 Measurement URL을 생성합니다. 같은 창에서 Click 대신 Impression을 선택하면 노출 URL을 작성할 수 있습니다.



2. 포스트백 URL 생성

  1. 파트너 메뉴에서 포스트백 URL을 선택합니다.

  2. 원하는 포스트백 이름을 선택합니다.

  3. 포스트백 템플릿 창에서 Vungle을 검색합니다.

  4. 앱의 플랫폼(iOS, Android 또는 Windows)을 선택합니다.

  5. TUNE의 템플릿은 모든 행에 대해 동일한 Vungle 앱 ID를 자동으로 채웁니다. 이를 무시하고 다른 플랫폼을 선택한 다음 계정 관리자로부터 받은 적절한 Vungle App ID를 입력할 수 있습니다.

  6. 템플릿 포스트백 URL을 확인 및 수정하려면 템플릿 수정을 클릭합니다. 매크로를 추가하거나 삭제할 수 있습니다.



3. 설치 후 이벤트 포스트백 설정

  1. 어트리뷰션 분석에 로그인합니다.



  2. 아직 수행하지 않은 경우 Vungle을 파트너로 통합하십시오.



  3. Vungle과 공유할 데이터를 선택하십시오.



  4. 이벤트를 선택한 후 각 이벤트 및 각 플랫폼에 대한 포스트백을 생성하십시오.



  5. 통합 파트너 탭에서 검색하여 나중에 더 PIE 데이터를 추가 할 수 있습니다.



조정

1. 클릭 URL 설정

  1. 앱의 조정 대시보드에서 All Settings를 선택합니다.
  2. Partner Setup으로 이동합니다.



  3. Vungle을 파트너로 추가하십시오.



  4. Vungle에서 받은 Vungle 앱 ID를 입력하십시오.



  5. 새 트래커를 생성하려면 Trackers ->를 선택하고 Vungle을 네트워크로 선택합니다. New Tracker를 클릭하고 트래커의 이름을 지정한 다음 Customize를 클릭합니다. 이렇게 하면 클릭 URL과 노출 URL이 생성되어 계정 관리자와 공유할 수 있습니다.

2. 설치 후 이벤트 포스트백 설정

  1. 수익 전달 및 링크 이벤트를 켭니다.



  2. 이벤트 이름을 Vungle에 전송하는 방식대로 지정하십시오.




Appsflyer

1. Vungle을 파트너로 설정 및 클릭 URL 설정

  1. 캠페인을 설정합니다.



  2. Vungle을 파트너로 구성합니다.





  3. 추적 URL을 생성합니다. Vungle 로고를 클릭하여 Vungle 구성 창을 엽니다.



  4. 캠페인 텍스트 필드에 캠페인의 이름을 입력합니다(선택 사항).

  5. Vungle과의 계약에 따라 클릭 어트리뷰션 룩백 창의 타임 프레임을 설정합니다.

  6. Vungle 계정 관리자로부터 받은 app_id를 삽입합니다.

  7. 활성화를 선택하면 모든 설치가 Vungle에 통보됩니다. 활성화 상자를 체크하면 통합이 완료되었음을 Vungle에 알립니다.

  8. 추적 링크를 복사하여 Vungle 계정 관리자에게 보냅니다.

  9. View-Through Attribution을 설정하려면 먼저 View-Through Attribution 사용 옵션을 선택한 다음 Vungle과의 계약에 따라 View-Through Attribution 룩백 창의 타임 프레임을 설정합니다.

  10. View-Through 추적 링크를 복사하여 Vungle 계정 관리자에게 보냅니다.



인앱 이벤트 구성

  1. Vungle 속성 설치 인앱 이벤트만 보낼지 또는 모든 인앱 이벤트를 보낼지 선택합니다.



  2. + 클릭하여 인앱 이벤트 매핑을 추가하고 Vungle에 보내려는 이벤트를 매핑합니다.



  3. 저장 및 닫기를 클릭합니다.

Kochava

클릭 URL 설정

  1. 앱을 열고 다음을 선택합니다. App Tools > Campaign Manager > Add a Campaign > Enter Campaign Name > Enter the Default Destination URL > Submit





  2. 클릭/노출 URL을 설정하려면 Edit Tracker를 선택하십시오. 클릭 및 노출 URL은 템플릿에 통합된 매크로를 기반으로 제공됩니다.

포스트백 URL 설정 및 설치 후 이벤트 공유

Kochava는 설치 후 이벤트를 Vungle에 보낼 수 있습니다. Vungle과 공유할 매개변수를 선택할 수 있습니다. 포스트백 URL을 설정하려면 다음을 선택합니다. App Tools > Partner Configuration > Custom Postback > Go > Set Event Status to Active (default) > Update Title > Destination URL> Set Delivery Delay to real time > Set number of Retry Attempts (up to 10)> Select Macro (s) to be passed within the Destination URL > Save





신규 통합

Vungle에 통합해주셔서 감사합니다! 여기에서 Vungle의 기본 API 자료를 보실 수 있습니다. 모든 설치를 다시 게시해주시기 바랍니다. 설치 후 이벤트를 Vungle에 보내는 방법을 자세히 알아보려면 이 페이지를 참조하십시오. 매크로는 다음과 같습니다.

Vungle 매크로

  • {{{id}}}: 각 광고 게재별 고유한 ID 24자리(포스트백에서 필요)
  • {{{app_id}}}: 시스템에서 앱을 식별하는 Vungle 앱 ID(포스트백에서 필요)
  • {{{campaign_id}}}: 시스템 내에서 캠페인을 식별하는 Vungle ID
  • {{{campaign_name}}}: 캠페인 이름
  • {{{creative}}}: 크리에이티브 이름
  • {{{ifa}}}: Apple의 광고주 식별자(포스트백에서 필요)
  • {{{aaid}}}: Google Play 서비스의 광고주 식별자(포스트백에서 필요)
  • {{{isu}}}: 폴백 식별자, iOS용 OpenUDID와 Android용 Android ID(Apple이 OpenUDID를 더 이상 사용하지 않아 선호도 하락)(포스트백에서 필요)
  • {{{site_id}}}: 소스를 식별하는 Vungle ID(광고가 재생된 앱)

View-Through Attribution 사용

대부분의 광고 네트워크에서 View-Through Attribution을 사용하므로 네트워크의 실적을 정확히 비교하기 위해 Vungle에서도 동일한 작업을 수행하는 것이 좋습니다. Vungle의 속성 창은 다음과 같습니다.

  • 14일 동안 클릭 수
  • 24시간 동안 노출 수
