Activation de l'attribution de view-through

TUNE

1. Configuration de l'URL de clic

  1. Ajoutez votre application à votre tableau de bord MAT.



  2. Pour des résultats optimaux, Vungle vous recommande de définir Attribution de view-through lorsque vous intégrez Vungle à TUNE. Si vous voulez le réactiver, faites-le sur la page Paramètres d'attribution.

  3. Créez l'URL de mesure en sélectionnant Clic. Dans la même fenêtre, vous pouvez créer l'URL d'impression en sélectionnant Impression au lieu de Clic.



2. Créez une URL de postback

  1. Dans le menu Partenaires, sélectionnez URL de postback.

  2. Choisissez le nom de postback souhaité.

  3. Recherchez Vungle dans la fenêtre Modèle de postback.

  4. Choisissez la plateforme de votre application : iOS, Android ou Windows.

  5. Le modèle de TUNE renseigne automatiquement le même ID d'application Vungle pour toutes les lignes. Vous pouvez ignorer ce processus et sélectionner une autre plateforme, puis indiquer l'ID d'application Vungle, que votre gestionnaire de compte vous a communiqué.

  6. Cliquez sur Modifier le modèle pour afficher et modifier l'URL de postback de modèle. Vous pouvez ajouter ou supprimer des macros.



3. Configuration de postback d'événements post-installation

  1. Connectez-vous à Analyse des attributions.



  2. Si ce n'est pas déjà fait, intégrez Vungle en tant que partenaire.



  3. Choisissez les données que vous souhaitez partager avec Vungle.



  4. Une fois que vous avez sélectionné les événements, créez des postbacks pour chaque événement et pour chaque plateforme.



  5. Vous pouvez ajouter plus de données PIE ultérieurement en effectuant une recherche dans l'onglet Partenaires intégrés.



adjust

1. Configuration de l'URL de clic

  1. Choisissez Tous Paramètres dans le tableau de bord adjust de votre application.



  2. Accédez à Configuration de partenaire.



  3. Ajoutez Vungle en tant que partenaire.



  4. Saisissez l'ID d'application Vungle que vous avez reçu de Vungle.



  5. Pour générer un nouveau suivi, sélectionnez Suivis -> choisissez Vungle en tant que réseau. Cliquez ensuite sur Nouveau suivi, nommez votre suivi, puis cliquez sur Personnaliser. Ceci crée une URL de clic et une URL d'impression, que vous pouvez communiquer à votre gestionnaire de compte.

2. Configuration de postbacks d'événements post-installation

  1. Activez Transmission de réseau et Événements de lien.



  2. Nommez les événements que vous souhaitez envoyer à Vungle.




Appsflyer

1. Configuration de Vungle en tant que partenaire et configuration de l'URL de clic

  1. Configurez la campagne.



  2. Configurez Vungle en tant que partenaire.





  3. Générez l'URL de suivi. Cliquez sur le logo Vungle pour ouvrir la fenêtre Configuration de Vungle.



  4. Saisissez le nom de votre campagne dans le champ de texte Campagne. Ceci est facultatif.

  5. Définissez l'intervalle de temps de la Fenêtre de retraçage d'attribution de clic conformément à votre contrat avec Vungle.

  6. Saisissez l'ID d'application que vous avez reçu de votre gestionnaire de compte Vungle.

  7. Lorsque vous sélectionnez Activer, Vungle est informé de toutes les installations. Une fois que vous cochez la case Activer, Vungle est informé que l'intégration est terminée.

  8. Copiez le lien de suivi et envoyez-le à votre gestionnaire de compte Vungle.

  9. Pour définir l'attribution de view-through, sélectionnez d'abord l'option Activer l'attribution de view-through, puis définissez l'intervalle de temps de la fenêtre Retraçage d'attribution de view-through conformément à votre contrat avec Vungle.

  10. Copiez le lien de suivi de view-through et envoyez-le à votre gestionnaire de compte Vungle.



Configuration d'événements intégrés à l'application

  1. Envoyez uniquement les événements intégrés à l'application d'installation attribués par Vungle ou envoyez tous les événements intégrés à l'application.



  2. Cliquez sur +Cliquer pour ajouter le mappage d'événements intégrés à l'application et mappez les événements que vous souhaitez envoyer à Vungle.



  3. Cliquez sur Enregistrer, puis sur Fermer.

Kochava

Configuration de l'URL de clic

  1. Localisez votre application > Sélectionnez Outils d'application > Gestionnaire de campagne > Ajouter une campagne > Saisissez le nom de la campagne > Saisissez l'URL de destination par défaut > Envoyez.





  2. Pour définir l'URL de clic/d'impression, sélectionnez Modifier le suivi. Les URL de clic et d'impression vous seront communiquées en fonction des macros intégrées au modèle.

Configuration d'URL de postback et partage d'événements post-installation

Kochava peut envoyer des événements post-installation à Vungle. Vous pouvez choisir les paramètres à communiquer à Vungle. Pour définir l'URL de postback, sélectionnez Outils d'application > Configuration de partenaire > Postback personnalisé > Aller > Définissez le statut d'événement sur Actif (valeur par défaut) > Mettre le titre à jour > URL de destination > Définissez le délai de livraison sur l'heure réelle > Définissez le nombre de nouvelles tentatives (jusqu'à 10) > Sélectionnez la ou les macros à transmettre dans l'URL de destination > Enregistrez.





Nouvelles intégrations

Merci d'avoir intégré Vungle ! Vous trouverez ici notre documentation d'API de base. Merci de bien vouloir nous retourner toutes les installations. Pour plus d'informations sur le processus consistant à nous envoyer des événements post-installation, reportez-vous à cette page. Nos macros sont les suivantes :

Macros Vungle

  • {{{id}}} : ID à 24 caractères unique à chaque diffusion de publicité (requis par le postback)
  • {{{app_id}}} : ID d'application Vungle identifiant votre application dans notre système (requis par le postback)
  • {{{campaign_id}}} : ID Vungle identifiant la campagne dans notre système
  • {{{campaign_name}}} : nom de la campagne
  • {{{creative}}} : nom de la création
  • {{{ifa}}} : identifiant Apple des publicitaires (requis par le postback)
  • {{{aaid}}} : identifiant de services Google Play des publicitaires (requis par le postback)
  • {{{isu}}} : identifiant de rechange, OpenUDID pour iOS et ID Android pour Android (en raison de la dépréciation forcée d'OpenUDID par Apple, il ne s'agit plus de la méthode préférée) (requis par le postback)
  • {{{site_id}}} : ID Vungle identifiant la source, à savoir l'application dans laquelle la publicité a été diffusée

Activation de l'attribution de view-through

Parce que l'attribution de view-through est activée pour la majorité des réseaux publicitaires, nous vous conseillons d'en faire de même afin que Vungle produise une comparaison précise des performances de notre réseau. Les fenêtres d'attribution de Vungle sont les suivantes :

  • 14 jours pour les clics
  • 24 heures pour les impressions
